DonationConnect — Files & Functions (brief)

This page lists the main files in the folder and the key UI features / JavaScript functions they provide. Click a file name to open it in the project folder.

project.html

  • Header / Hero UI and site navigation.
  • Top-right auth CTA (Login / Sign-up) — simulated behavior only.
  • Hero: "Get Started" button disabled (visual only).
  • Added DonationConnect description block and "Learn More" link.
  • JS:
    • createBtn handler — creates a demo donation and stores it in localStorage under donationconnect:donations, then redirects to track.html?id=....
    • Sidebar tab switching logic for inline dashboard sections (if present) and hash-based activation.
    • Utility highlightTopNav() — marks top nav active.

dashboard.html

  • Platform dashboard moved to its own page.
  • Sidebar with four tabs: Overview, Donation Drives, Reports & Analytics, Modules.
  • Main content sections:
    • Overview — cards for Active Users, Active Donations, Pending Deliveries.
    • Donation Drives — lists drives saved in localStorage key donationconnect:drives.
    • Reports — basic analytics rendered from stored data.
    • Modules — buttons opening module pages.
  • JS functions:
    • setActiveTab(tabId) — activates sidebar tab and triggers render as needed.
    • renderDrives() — renders drives list from localStorage.
    • renderReports() — simple reports (totals, by-status).
    • renderOverview() — fills overview stats.
    • readJSON(key), escapeHtml(s).
    • Drive actions: editDrive(id), deleteDrive(id).
    • Demo donation creation similar to project.html (uses donationconnect:donations).

create-drive.html

  • Form for creating a new Donation Drive.
  • JS:
    • uuidv4() — generates an id.
    • Form submit handler — validates required fields, saves drive to localStorage under donationconnect:drives, then redirects back to dashboard.html.
    • Cancel button — returns to dashboard.

module.html (Modules index)

  • Modules index UI: Admin, Donor, Recipient, Logistics cards with descriptions and "Open" links.
  • Search: input with client-side filter — hides/shows module cards by title.
  • Styling: professional layout and responsive grid.

Module pages

  • admin-module.html
    • Admin module shell (id=admin) with forms and professional CSS styling.
    • Styles include form controls, tables and utility classes; no heavy JS included in file.
  • donor-module.html
    • Donor UI shell (id=donor) with form layout, inputs, and polished controls.
    • CSS focuses on readable forms and responsive behavior; no page-level JS required.
  • recipient-module.html
    • Recipient UI shell (id=recipient) with form controls, textareas, and buttons.
  • logistics-module.html
    • Logistics UI (id=logistics) including:
      • Assign Delivery select(s) — options can be shown one-per-line using size attribute or as normal selects.
      • Delivery Agent Name input, Delivery Status select and Update button (form layout ensures one control per row with spacing).
      • Professional CSS for panels, map placeholder and lists.

Other notes

  • Local storage keys used:
    • donationconnect:drives — stores drives created via create-drive.
    • donationconnect:donations — stores donations created by demo flows.
  • Many pages include shared CSS patterns (variables, responsive grid, card styles).
